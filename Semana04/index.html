Dia 01

Paradigmas

Toda linguagem de programação possui um paradigma.

Se encontra dentro de alguma classificação conforme a suas funcionalidades e do modo como permite que nós estruturemos o nosso código.

O javascript (de diversas outras linguagens) possui vários paradigmas.

  • Orientado a objetos;
  • Orientado a eventos;
  • Funcional e outras;
POO

POO vem de Programação Orientada a Objetos, tambem conhecida como OOP (Object Oriented Programming)

Extrair, dentro do codigo, objetos do mundo real.

Permite que os objetos interajam entre si através da troca de mensagens.

A base da POO são as Classes e os Objetos.


Todo objeto é criado a partir de uma classe.

A classe é um molde que é utilizado para criar um objeto.

Dentro da classe nos conseguimos definiro os atributos e métodos que um objeto receberá.

Construtores

Instância de uma classe é o que da origem a um objeto.

A classe vem com um metodo especial que chamamos de construtor. Utilizamos um construtor para definir valores iniciais dos atributos de um objeto. Esse método é chaamdo quando criamos uma instância de uma classe com a palavra.

Herança

É um recurso do Paradigma Orientado a Objetos.

Permite que classes herdem caracteristicas de outras classes, isto é, declaramos uma classe que vai herdar metodos e atributos de outra classe.

é uma técnica de reuso de código.

As duas classes devem ter uma relação de É UM.

Como exemplo tem-se: Todo capitão É UMA pessoas. Poderiamos ter uma classe Captain herdando de Person.

Dia 03

Exceções

Exceções são erros que ocorrem no código.

São capturados em tempo de execução(runtime).

Podem ser tratadas para que o código nao quebre.

No javascript, as exceções são lançadas pela classe Error.

Podemos utilizar o try e catch para poder tratar estes erros.

Podemos lançar nossos próprios erros atraves do throw.
